# 实施任务清单

## 任务概述

修复首页"立即安装"按钮下拉菜单的 z-index 层级问题，确保下拉菜单正确显示在活动指标组件之上。

## 任务列表

### 1. 代码修改

**任务 ID:** TASK-001
**标题:** 修改下拉菜单 z-index 值
**文件:** `src/components/home/InstallButton.module.css`
**工作量:** 5 分钟

**具体步骤:**
1. 打开 `src/components/home/InstallButton.module.css`
2. 定位到 `.dropdownMenu` 样式规则（约第 142 行）
3. 将 `z-index: 1000;` 修改为 `z-index: 9999;`
4. 保存文件

**修改内容:**
```css
.dropdownMenu {
  position: absolute;
  top: calc(100% + 0.25rem);
  left: 0;
  min-width: 240px;
  background: var(--color-surface);
  background-color: #ffffff;
  border: 1px solid var(--color-border);
  border-color: rgba(0, 0, 0, 0.1);
  border-radius: 0.5rem;
  box-shadow: var(--shadow-lg);
  box-shadow: 0 10px 40px rgba(0, 0, 0, 0.15);
  list-style: none;
  margin: 0;
  padding: 0.5rem;
  opacity: 0;
  visibility: hidden;
  transform: translateY(-10px);
  transition: all 0.2s ease-out;
  z-index: 9999; /* 从 1000 修改为 9999 */
}
```

**验收标准:**
- [ ] 代码修改完成
- [ ] 无语法错误

## 风险缓解

| 风险 | 缓解措施 |
|------|---------|
| z-index 过高可能影响其他组件 | 使用 9999 而非更高值，这是常见的"最大值"约定 |
| 层叠上下文限制 | 验证父容器不限制层级，必要时调整 |
