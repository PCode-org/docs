# SEO 维护检查清单

本文档提供 SEO 的定期维护检查清单,确保站点的 SEO 配置始终保持最佳状态。

## 新页面发布检查清单

在发布新页面或博客文章之前,使用此清单确保 SEO 基础配置正确:

### 内容检查

- [ ] **页面有唯一的标题**
  - 标题准确描述页面内容
  - 标题长度在 50-60 个字符之间
  - 标题包含目标关键词(如有)

- [ ] **页面有唯一的描述**
  - 描述准确概括页面内容
  - 描述长度在 150-160 个字符之间
  - 描述包含目标关键词(如有)

- [ ] **内容质量高**
  - 内容原创且有价值
  - 没有拼写和语法错误
  - 格式清晰,易于阅读

### 技术检查

- [ ] **URL 结构合理**
  - URL 简短且描述性强
  - 使用小写字母和连字符
  - 避免使用特殊字符和参数

- [ ] **图片优化**
  - 所有图片都有 alt 属性
  - 图片文件大小已优化
  - 使用适当的图片格式(WebP 优先)

- [ ] **内部链接**
  - 页面包含相关内部链接
  - 链接使用描述性锚文本
  - 没有损坏的链接

## 每月 SEO 维护检查清单

建议每月执行一次以下检查:

### Sitemap 和 Robots.txt

- [ ] **验证 Sitemap 可访问**
  ```bash
  curl -I https://hagicode-org.github.io/site/sitemap-index.xml
  ```
  - 返回状态码应为 200
  - 内容类型应为 XML

- [ ] **检查 Sitemap 内容**
  - 打开 sitemap-index.xml
  - 确认所有重要页面都包含在内
  - 检查是否有需要删除的页面

- [ ] **验证 Robots.txt**
  ```bash
  curl https://hagicode-org.github.io/site/robots.txt
  ```
  - 确认语法正确
  - 确认没有错误地阻止重要页面

### 搜索引擎控制台

#### Google Search Console

- [ ] **检查覆盖率报告**
  - 查看"错误"部分是否有新增问题
  - 查看"有效但有警告"部分
  - 修复发现的错误并重新验证

- [ ] **查看效果报告**
  - 检查搜索点击率(CTR)
  - 查看排名靠前的关键词
  - 识别表现差的页面并优化

- [ ] **检查手动操作**
  - 查看是否有手动处罚
  - 如有,按照指南修复并申请复审

#### Bing Webmaster Tools

- [ ] **查看 SEO 报告**
  - 检查是否有 SEO 建议
  - 修复发现的 SEO 问题

- [ ] **检查爬虫错误**
  - 查看是否有爬虫无法访问的页面
  - 修复 404 和 5xx 错误

### Meta 标签验证

- [ ] **测试 Open Graph 标签**
  - 使用 [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/)
  - 输入重要页面 URL
  - 验证标题、描述和图片显示正确

- [ ] **测试 Twitter Card**
  - 使用 [Twitter Card Validator](https://cards-dev.twitter.com/validator)
  - 输入重要页面 URL
  - 验证卡片显示正确

- [ ] **验证结构化数据**
  - 使用 [Google Rich Results Test](https://search.google.com/test/rich-results)
  - 输入重要页面 URL
  - 修复所有错误和警告

### 性能检查

- [ ] **测试页面加载速度**
  - 使用 [Google PageSpeed Insights](https://pagespeed.web.dev/)
  - 目标: 移动端和桌面端分数都超过 80

- [ ] **检查 Core Web Vitals**
  - LCP (最大内容绘制) < 2.5s
  - FID (首次输入延迟) < 100ms
  - CLS (累积布局偏移) < 0.1

## 每季度 SEO 审查检查清单

建议每季度执行一次全面审查:

### 内容审查

- [ ] **审查流量趋势**
  - 使用 Google Analytics 分析流量变化
  - 识别流量下降的页面并找出原因

- [ ] **更新过时内容**
  - 审查发布超过 6 个月的内容
  - 更新过时信息
  - 添加新的相关内容

- [ ] **关键词分析**
  - 查看目标关键词的排名变化
  - 识别新的关键词机会
  - 优化内容以提升排名

### 技术审查

- [ ] **检查损坏的链接**
  ```bash
  # 使用工具检查整个站点
  # 或使用在线工具如 Dead Link Checker
  ```
  - 修复或删除损坏的内部链接
  - 更新或移除损坏的外部链接

- [ ] **审查站点结构**
  - 确保站点层级清晰(不超过 3-4 层)
  - 检查重要页面是否可以从首页在 3 次点击内到达
  - 优化导航结构

- [ ] **移动友好性测试**
  - 使用 [Google Mobile-Friendly Test](https://search.google.com/test/mobile-friendly)
  - 修复移动体验问题

### 竞争对手分析

- [ ] **分析竞争对手**
  - 识别主要竞争对手
  - 查看他们的关键词策略
  - 学习他们的最佳实践

## 年度 SEO 策划检查清单

建议每年进行一次全面的 SEO 策划:

- [ ] **回顾年度目标**
  - 检查 SEO KPI 是否达成
  - 分析成功和失败的原因
  - 制定明年的目标

- [ ] **更新 SEO 策略**
  - 根据搜索引擎算法更新调整策略
  - 评估新技术的应用(如 AI 搜索)
  - 更新关键词策略

- [ ] **培训和学习**
  - 学习最新的 SEO 最佳实践
  - 参加 SEO 相关培训和会议
  - 更新团队 SEO 知识

## 紧急问题响应

当出现以下紧急情况时,立即采取行动:

### 搜索流量突然下降

- [ ] **检查 Google Search Console**
  - 查看是否有手动处罚
  - 检查覆盖率报告是否有大量错误

- [ ] **检查站点可访问性**
  - 确认站点在线
  - 测试重要页面是否可访问
  - 检查 robots.txt 是否被误修改

- [ ] **查看算法更新**
  - 查看是否有最近的搜索引擎算法更新
  - 分析更新对站点的影响
  - 调整策略以适应新算法

### 大量页面被移除索引

- [ ] **确认 robots.txt 配置**
  - 检查是否错误地阻止了爬虫
  - 修复并重新提交 sitemap

- [ ] **检查 noindex 标签**
  - 确认重要页面没有 noindex 标签
  - 移除错误的 noindex 配置

- [ ] **提交重新索引请求**
  - 对受影响的页面请求重新编入索引
